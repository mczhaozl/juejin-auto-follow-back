# 千问 Qwen 负责人离职：林俊旸卸任与团队变动（2026 年 3 月）

> 梳理阿里千问大模型技术负责人林俊旸及多名核心成员离职的时间、来源与背景，以及对开源/API 使用者的影响与建议。

---

## 一、变动概览

2026 年 3 月上旬，**阿里通义千问（Qwen）大模型技术负责人林俊旸**及多名技术骨干先后离职，引发业内关注。本文按「时间线 + 来源 + 对开发者的影响」整理，便于你判断对 Qwen 开源与 API 使用的影响。

---

## 二、时间线与主要人员

### 林俊旸卸任

- **时间**：2026 年 **3 月 4 日**凌晨（据财新、新浪财经、每日经济新闻等报道）。
- **方式**：林俊旸在 **X（原 Twitter）** 发文宣布卸任，内容大意为「me stepping down. bye my beloved qwen」（我卸任了，再见了我亲爱的千问）。
- **背景**：林俊旸为阿里达摩院通义千问系列大模型**技术负责人**，曾被报道为阿里「最年轻 P10」之一，2022 年底起负责千问技术线。

### 其他离职人员（据公开报道）

多家媒体同期报道，与千问相关的多名技术骨干离职或调整，包括但不限于：

- **郁博文**：Qwen 后训练负责人；
- **惠彬原**：代码模型负责人；
- **Kaixin Li**：Qwen3.5 及视觉编程相关开发者。

具体名单与职责以各公司及当事人后续信息为准。

## 三、报道中的背景与原因（来源说明）

以下整理自**财新、新浪财经、每日经济新闻、网易等**的报道，供参考，不代表官方定论。

### 1. 组织架构调整

报道称，阿里计划将千问团队从**垂直整合**（预训练、后训练、多模态、基础设施等由同一技术线负责）改为**水平分工**（拆成预训练、后训练、文本、多模态等独立团队）。林俊旸的管理范围随之收缩，据称其主张「预训练、后训练、Infra 与训练应更紧密协同」，与上述拆分方向存在分歧。

### 2. 人事与战略方向

报道提及**前 Google DeepMind 高级研究员、曾参与 Gemini 项目的 Hao Zhou 加入千问**，担任新的技术负责人。部分报道将此事与「从开源为主转向闭源/商业化」的猜测相联系；阿里与千问官方未在本文撰写前对「开源是否会收缩」做明确表态。

### 3. 开源与商业化

部分分析认为，阿里 AI 战略正从「比模型、拼开源影响力」转向「拼体系、与电商/支付等业务融合」，模型从「外挂能力」升级为「用户服务主界面」。此类调整可能与坚持开源、社区优先的团队思路产生张力。

## 五、对开发者的影响与建议

### 1. 开源模型与社区

- **现状**：截至本文撰写时，**Qwen 系列在魔搭（ModelScope）、HuggingFace、GitHub 上的开源仓库与模型仍可正常访问与下载**，未出现「下架」或「停止维护」的官方公告。
- **建议**：若你依赖 Qwen 开源版本做本地部署或二次开发，可**持续关注阿里通义/魔搭、Qwen 官方仓库的公告与 Release**；对关键项目建议保留模型权重与依赖版本的本地备份。

### 2. API 与商业服务

- **现状**：阿里云百炼、通义等提供的 **Qwen 系列 API 与产品线** 由阿里云/业务方运营，负责人变动属于团队与组织层面，**不直接等同于 API 关停或合同变更**。
- **建议**：已接入 API 的团队**按现有合同与 SLA 使用**即可；若有长期合约或关键业务依赖，可主动与阿里云/商务确认产品路线图与支持策略。

### 3. 后续技术路线

- 新负责人上任后，**功能优先级、开源节奏、模型命名与版本策略**可能逐步调整；若有新公告或博客，建议以**阿里通义、魔搭、GitHub Qwen 官方仓库**为准。
- 对「是否会减少开源、转向闭源」等猜测，目前**无官方明确说法**，建议以官方公告为准，不依赖单篇报道做重大架构决策。
