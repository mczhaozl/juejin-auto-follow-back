# TCP/IP 完全指南：网络层协议与通信原理

> 深入讲解 TCP/IP 协议栈，包括 TCP/UDP 区别、三次握手、四次挥手、拥塞控制，以及实际网络问题排查和性能优化。

## 一、TCP/IP 协议栈

### 1.1 四层模型

```
┌─────────────────────────────────────────────────────────────┐
│                    TCP/IP 协议栈                             │
│                                                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 应用层    HTTP / FTP / DNS / SSH                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                         │                                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 传输层    TCP / UDP                                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                         │                                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 网络层    IP / ICMP / ARP                           │   │
│  └─────────────────────────────────────────────────────┘   │
│                         │                                    │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 链路层    Ethernet / Wi-Fi                          │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 数据封装

```
应用层数据
    │
    ▼
传输层 (TCP) + 端口
    │
    ▼
网络层 (IP) + IP 地址
    │
    ▼
链路层 (Ethernet) + MAC 地址
```

## 二、TCP vs UDP

### 2.1 对比

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠 | 不可靠 |
| 速度 | 慢 | 快 |
| 头部 | 20-60 字节 | 8 字节 |
| 场景 | HTTP、邮件 | DNS、视频 |

### 2.2 TCP 特点

```c
// TCP 特性
- 面向连接
- 可靠传输（确认、重传）
- 流量控制（滑动窗口）
- 拥塞控制（慢启动、拥塞避免）
- 全双工通信
```

### 2.3 UDP 特点

```c
// UDP 特性
- 无连接
- 不保证可靠
- 无重传
- 无流量控制
- 轻量级
- 实时性高
```

## 三、三次握手

### 3.1 握手过程

```
┌─────────────────────────────────────────────────────────────┐
│                      三次握手                                │
│                                                              │
│  客户端                              服务器                  │
│    │                                   │                    │
│    │─────── SYN=1, seq=x ─────────────▶│                    │
│    │                                   │                    │
│    │◀───── SYN=1, ACK=1, seq=y,       │                    │
│    │        ack=x+1                    │                    │
│    │                                   │                    │
│    │─────── ACK=1, seq=x+1,           │                    │
│    │        ack=y+1 ─────────────────▶│                    │
│    │                                   │                    │
│    │         连接建立完成              │                    │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 为什么需要三次

```c
// 目的：同步双方的序列号
// 1. 客户端发送 SYN，服务器确认收到
// 2. 服务器发送 SYN+ACK，客户端确认收到
// 3. 客户端发送 ACK，服务器确认收到
// 结果：双方都确认对方的序列号
```

## 四、四次挥手

### 4.1 挥手过程

```
┌─────────────────────────────────────────────────────────────┐
│                      四次挥手                                │
│                                                              │
│  客户端                              服务器                  │
│    │                                   │                    │
│    │─────── FIN=1, seq=u ─────────────▶│                    │
│    │                                   │                    │
│    │◀───── ACK=1, ack=u+1             │                    │
│    │                                   │                    │
│    │         (服务器继续发送数据)      │                    │
│    │                                   │                    │
│    │◀───── FIN=1, seq=v ─────────────│                    │
│    │                                   │                    │
│    │─────── ACK=1, ack=v+1 ─────────▶│                    │
│    │                                   │                    │
│    │         连接关闭完成              │                    │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 状态变化

```
客户端:  FIN_WAIT_1 → FIN_WAIT_2 → TIME_WAIT → CLOSED
服务器:  CLOSE_WAIT → LAST_ACK → CLOSED
```

## 五、流量控制

### 5.1 滑动窗口

```
┌─────────────────────────────────────────────────────────────┐
│                    滑动窗口                                  │
│                                                              │
│  发送窗口: [ 1, 2, 3, 4, 5, 6, 7, 8 ]                      │
│              ▲    ▲         ▲                               │

│              ▲    ▲                                         │

```

### 5.2 机制

```c
// 流量控制
receiver: 发送窗口大小
sender:   根据窗口控制发送

// 窗口为0时
sender: 探测报文
receiver: 更新窗口大小
```

## 六、拥塞控制

### 6.1 算法

```
┌─────────────────────────────────────────────────────────────┐
│                   拥塞控制                                   │
│                                                              │
│  慢启动 ──▶ 拥塞避免 ──▶ 拥塞发生 ──▶ 快速恢复              │
│    │           │              │           │                │
│   cwnd=1    cwnd=MAX      cwnd=1      cwnd=MAX/2         │
│   指数增长   线性增长      乘法减少    快速恢复             │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 概念

| 概念 | 说明 |
|------|------|
| cwnd | 拥塞窗口 |
| ssthresh | 阈值 |
| RTT | 往返时间 |
| ACK | 确认 |

### 6.3 阶段

```c
// 慢启动
cwnd = 1
每收到一个 ACK，cwnd++（指数增长）

// 拥塞避免
cwnd 达到阈值后
每经过一个 RTT，cwnd++（线性增长）

// 拥塞发生
超时: cwnd = 1, ssthresh = cwnd/2
快重传: cwnd = cwnd/2, ssthresh = cwnd/2
```

## 七、网络排查

### 7.1 常用命令

```bash
# 查看网络连接
netstat -tuln          # 监听端口
netstat -an            # 所有连接

# 替代 netstat
ss -tuln

# 网络路径
traceroute example.com # 路由追踪（UDP）
tracert example.com    # Windows

# 连通性测试
ping example.com

# DNS 查询
dig example.com
nslookup example.com
```

### 7.2 TCPdump

```bash
# 捕获所有包
tcpdump -i eth0

# 捕获特定端口
tcpdump port 80

# 捕获特定主机
tcpdump host 192.168.1.1

# 捕获并保存
tcpdump -w capture.pcap
```

## 八、总结

TCP/IP 核心要点：

2. **TCP vs UDP**：可靠 vs 高效
3. **三次握手**：建立连接
4. **四次挥手**：关闭连接
5. **流量控制**：滑动窗口
6. **拥塞控制**：慢启动/拥塞避免
7. **排查**：netstat/tcpdump

掌握这些，网络通信 so easy！
