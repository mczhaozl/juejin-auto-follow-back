# AI 提示工程完全指南：从基础到高级技巧

提示工程是与 AI 大模型交互的关键技能。本文将带你全面掌握提示工程的各种技巧。

## 一、提示工程基础

### 1. 什么是提示工程

提示工程是设计和优化提示词，以获得 AI 最佳输出的过程。

### 2. 基础提示结构

```
角色: 你是一个资深的前端开发专家
任务: 帮我优化这段代码
背景: 这是一个 React 组件，用于显示用户列表
要求:
  1. 提高性能
  2. 增加可维护性
  3. 添加 TypeScript 类型
```

### 3. 清晰明确的指令

```
❌ 不好的提示:
写点代码

✅ 好的提示:
你是一个 Python 专家。请写一个函数，输入是一个整数列表，
输出是列表中的最大整数。要求:
1. 时间复杂度 O(n)
2. 处理空列表的情况
3. 添加完整的文档字符串
```

## 二、提示工程核心原则

### 1. 明确角色

```
你是一个资深的技术作家，擅长将复杂的技术概念用通俗易懂的语言解释清楚。
你有 10 年的技术写作经验，为多个知名技术博客撰稿。
```

### 2. 提供上下文

```
我正在开发一个电商网站的购物车功能。
用户可以添加商品、修改数量、删除商品。
现在需要实现价格计算功能，包括：
- 商品小计
- 优惠券折扣
- 运费计算
- 总价

请帮我设计这个功能的代码结构。
```

### 3. 具体要求

```
请写一篇关于 React Hooks 的文章，要求：
1. 字数 2000 字以上
2. 包含 useState、useEffect、useContext、useReducer 的详细讲解
3. 每个 Hook 至少 2 个实战例子
4. 代码注释详细
5. 适合中级开发者阅读
```

### 4. 指定输出格式

```
请用 JSON 格式输出，包含以下字段：
- title: 文章标题
- summary: 100 字摘要
- tags: 标签数组
- content: 文章内容（Markdown 格式）

输出示例：
{
  "title": "React Hooks 入门",
  "summary": "本文介绍 React Hooks 的基本用法...",
  "tags": ["React", "Hooks", "前端"],
  "content": "# React Hooks 入门\n\n..."
}
```

## 三、高级提示技巧

### 1. 思维链 (Chain of Thought)

```
请一步步解决这个问题：
问题: 一个水池，单独用 A 管注水需要 6 小时，单独用 B 管注水需要 4 小时。
同时用两个管注水需要多少小时？

请先写出你的思考过程，再给出答案。
```

### 2. 少样本学习 (Few-shot Learning)

```
请根据以下例子，继续完成：

输入: "Hello" → 输出: "HELLO"
输入: "World" → 输出: "WORLD"
输入: "TypeScript" → 输出: ?
```

### 3. 角色换位

```
你现在是一个严格的代码审查专家。请审查以下代码，找出所有问题：
1. 潜在的 bug
2. 性能问题
3. 代码风格问题
4. 安全问题

对于每个问题，给出：
- 问题描述
- 严重程度（高/中/低）
- 修改建议

代码：
[在此粘贴代码]
```

### 4. 分步提示

```
我们来分步骤完成这个任务：

步骤 1: 分析需求，列出功能点
步骤 2: 设计数据结构
步骤 3: 设计 API 接口
步骤 4: 实现核心逻辑
步骤 5: 添加错误处理
步骤 6: 编写测试用例

让我们从步骤 1 开始。
```

## 六、调试提示

### 1. 让 AI 解释输出

```
我不理解这个代码的执行流程。请一步步解释：
1. 函数调用顺序
2. 每次调用时的变量值
3. 最终返回值是如何得到的

代码：
function fib(n) {
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2);
}
console.log(fib(5));
```

### 2. 让 AI 自我检查

```
请写一个函数求数组的平均值。

写完后，请：
1. 检查代码是否有 bug
2. 测试几个边界情况
3. 如果有问题，修正它
```

### 3. 对比分析

```
请对比以下两种实现方式：

实现 A:
[代码 A]

实现 B:
[代码 B]

请从以下角度对比：
1. 性能
2. 可读性
3. 可维护性
4. 内存使用
5. 适用场景
```
