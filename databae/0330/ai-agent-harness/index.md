# Harness 到底是什么？一篇大白话讲清楚 AI Agent 的执行框架

> 用最通俗的比喻讲清楚 AI Agent 的核心组件——Harness，帮你理解为什么模型强不等于 Agent 强。

## 一、先说结论

**Agent harness**，本质上就是：

> 一套“把模型 + 工具 + 流程 + 状态管理”组织起来，让 Agent 能稳定运行的执行框架。

如果你只记一句话：

> **模型决定能力，Harness 决定能不能用。**

## 二、为什么会出现 harness 这个东西？

很多人一开始做 AI Agent，路径是这样的：

1. 写 prompt
2. 接一个 LLM API
3. 加一点工具调用（function calling / tools）
4. 跑起来了

看起来已经是 Agent 了。

但很快你会遇到这些问题：

- 有时候能跑，有时候崩
- 上下文丢了
- 工具调用顺序混乱
- 一次任务中断后无法恢复
- 多步任务很难控制
- 不知道 agent 现在“在干嘛”

这时候你会发现：

👉 **你不是缺模型能力，你是缺“控制系统”**

而 harness 就是这个控制系统。

## 三、用一个最通俗的比喻

把 Agent 想象成一个“实习生”。

- **LLM** = 大脑（会思考）
- **Tools** = 工具（浏览器、代码执行、数据库等）
- **Prompt** = 指令

那 harness 是什么？

👉 **是“管理这个实习生的那套流程 + 制度”**

比如：

- 什么时候让他思考？
- 什么时候调用工具？
- 调错了怎么办？
- 任务做一半能不能接着做？
- 做完了怎么判断结束？

没有 harness，相当于：

👉 **给一个聪明人 + 一堆工具 + 一句“你去干活吧”**

结果一定混乱。

## 四、一个具体例子

假设你要做一个 Agent：

👉 “帮我分析一个网站，然后输出商业分析报告”

### ❌ 没有 harness 的版本

流程是这样的：

1. Prompt：去分析这个网站
2. LLM：决定调用浏览器工具
3. 抓取页面
4. 再调用 LLM 分析

问题来了：

- 如果网页太大怎么办？
- 分析一半 token 不够怎么办？
- 如果网页加载失败？
- 如果需要多次浏览？
- 如果要保存中间结果？

👉 **全靠你在 prompt 里“赌”**

### ✅ 有 harness 的版本

harness 会帮你做这些：

**1. 任务拆分**

- step1：抓首页
- step2：提取结构
- step3：补充更多页面
- step4：生成报告

**2. 状态管理**

- 已抓取哪些页面
- 当前进度
- 中间结果缓存

**3. 工具调度**

- 什么时候用 browser
- 什么时候用 summarizer
- 是否重复调用

**4. 错误处理**

- 页面失败 → retry
- token 超限 → 分块处理

**5. 循环控制（非常关键）**

- 是否继续探索？
- 是否结束？

👉 **这整套东西，就是 harness。**

## 五、从结构上看，Harness 包含什么？

可以把它拆成 5 个核心模块：

### 1. 控制循环（Control Loop）

这是 Agent 的“心跳”：

> 思考 → 决策 → 调用工具 → 更新状态 → 再思考

没有这个，你就不是 agent，只是一次调用。

### 2. 状态管理（State / Memory）

比如：

- 当前任务进度
- 已经做过的事情
- 中间输出

👉 **没这个，Agent 会“失忆”**

### 3. 工具编排（Tool Orchestration）

不是“能用工具”，而是：

👉 **什么时候用、用几次、用哪个**

### 4. 任务分解（Planning）

复杂任务必须拆：

- planner（规划）
- executor（执行）

很多 Agent 不稳定，本质是：

👉 **没有明确的 plan 层**

### 5. 错误恢复（Recovery）

现实世界一定会失败：

- API timeout
- tool 出错
- hallucination

Harness 要负责：

👉 **“兜底”**

## 六、为什么说 Harness 决定上限？

因为现在的趋势是：

👉 **模型能力在趋同（GPT / Claude / Gemini）**

真正的差异在：

- 谁能跑更复杂任务
- 谁能长期运行
- 谁的 Agent 更稳定

而这些：

👉 **都是 Harness 的问题**

## 七、和你现在做的东西有什么关系？

你现在如果在做：

- AI workflow
- 自动化 agent
- coding agent（比如 codex / claude code）
- AI 产品功能

你其实已经在“隐式写 harness”

只是可能：

- 写在 prompt 里
- 写在 if/else 里
- 写在 glue code 里

但随着复杂度上升：

👉 **这些会崩**

你迟早要：

👉 **把 harness 抽出来，变成“第一等公民”**

## 八、一个更直白的总结

如果你还没完全理解，我换一个更极简的说法：

👉 **Harness = Agent 的操作系统**

- **LLM** = CPU
- **Tools** = 外设
- **Harness** = OS

没有 OS：

👉 **再强的 CPU 也跑不起来**
