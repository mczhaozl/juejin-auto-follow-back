# 小红书笔记生成与发布

基于「选题专家 + **标题专家** + 内容专家 + 风格专家 + 审美专家 + 小红薯运营专家 + 发布专家」生成并发布符合小红书习惯的笔记。**标题由标题专家专门产出，不解析正文里的 `# 标题`。** 发布流程使用 **redbook/** 下脚本（语义化 DOM、标题≤20 字、正文简短、话题、模板）。

## 何时使用

- 用户说：写一篇小红书笔记、生成种草文案、帮我发小红书、小红书选题/**标题**/内容/风格/运营/审美/代发
- 需要：选题、**专门产出发布用标题（标题专家）**、笔记体结构、模板选择、运营向内容、代为执行发布

---

## 一、选题专家 (Topic Selection Expert)

**目标**：选出有话题感、易传播、适合小红书受众的题目。

### 选题原则

1. **话题感**：能引发「我也是」「学到了」「想试试」的共鸣或好奇。
2. **受众**：年轻、偏女性、生活/美妆/学习/职场/搞钱等场景，一句话说清「写给谁」。
3. **可视觉化**：能联想到封面图、清单、对比、前后效果等，便于配图。
4. **标题吸睛**：数字、痛点、结果、反常识、问句等，控制在 20 字内更易完读。

### 选题产出

- **主标题**（建议 15–20 字，含 1 个核心关键词或数字/问句）
- **一句话价值**（读者能得到什么 / 解决什么）
- **目标读者**（场景 + 身份）
- **3–5 个可选角度**（选一个为主线，其余可作小标题或后续笔记）

### 选题检查

- [ ] 标题有话题感或关键词
- [ ] 能在一句话内说清「问题/场景 + 收获」
- [ ] 适合配图或清单式呈现

---

## 二、标题专家 (Title Expert)

**目标**：**专门**产出用于发布的笔记标题，不解析正文里的 `# 标题`。标题专家只做这一件事。

### 职责与产出

- **产出**：一条发布用标题，**最多 20 字**（平台限制），含 1 个核心关键词或数字/问句/痛点。
- **原则**：吸睛、可扫读、与正文和选题一致；不从 md 文件首行 `# xxx` 解析，由标题专家根据选题与正文重新拟定。
- **交付**：发布时由发布专家将标题专家产出的标题通过 `--title "标题"` 传入脚本；脚本**不再**从文件解析标题。

### 标题专家检查

- [ ] 标题 ≤20 字
- [ ] 含关键词或数字/问句/结果感
- [ ] 未使用「解析正文 # 标题」的方式，而是由标题专家单独产出

---

## 三、内容专家 (Content Expert)

**目标**：把选题拆成小红书常见的「笔记体」结构，信息密度高、易扫读、可跟做。

### 笔记体结构建议

```markdown
# [吸睛标题]

> 一句话：读者能得到什么 / 为什么值得看

## 一、背景 / 痛点（可选）
- 1–2 句场景或问题，引发共鸣

## 二、核心内容（主体）
- 分点写，每点 1–2 句；可带小标题
- 清单、步骤、对比、前后效果等
- 适当用 emoji 做视觉分隔（不要过多）

## 三、总结 / 行动建议
- 3 条以内要点或下一步动作
- 可加互动语（收藏、关注、评论区见）
```

### 内容原则

- **短句**：一段不超过 2–3 行，多用列表。
- **正文≤1000 字**：小红书限制，超出会被打回；发布脚本会自动截断，内容专家宜控制篇幅或拆成多篇。
- **可操作**：步骤、清单、链接、关键词可复制。
- **真实感**：避免纯广告腔，可带个人体验或踩坑。
- **话题标签**：文末可建议 3–5 个 # 话题，与内容匹配；发布时需携带话题（见发布专家）。

### 内容检查

- [ ] 结构清晰、小标题能扫读
- [ ] 无长段堆砌，列表/分点明确
- [ ] 有 1 处以上「可跟着做」或「可收藏」的价值点

---

## 三、风格专家 (Style Expert)

**目标**：统一小红书常见的语气与排版，提升完读与互动感。

### 语气与人称

- **推荐**：亲切、口语化一点（「你」「咱们」「真的」「绝了」适度使用）；可带一点情绪词，但不过度夸张。
- **避免**：过于正式、长句、标题党（震惊体、绝对化承诺）。

### 排版规范

- **标题**：一级标题 = 笔记标题；二级 `##` 为块；三级 `###` 为小节（可选）。
- **列表**：多用无序列表；有序列表用于步骤。
- **强调**：**加粗**关键结论或数字，少用全段加粗。
- **话题**：文末用 `#话题一 #话题二` 形式建议标签。

### 风格检查

- [ ] 语气统一、易读
- [ ] 无错别字、标点统一
- [ ] 小标题能串联成「迷你提纲」

---

## 四、审美专家 (Aesthetic Expert)

**目标**：为笔记选择**模板/样式**，提升视觉与点击率。发布时需选择模板（如 素雅底纹、大图纯享、简约白 等）。

### 产出

- **推荐模板**：根据内容类型与受众推荐 1 个（如干货/清单 → 素雅底纹；强视觉/大图 → 大图纯享）。
- **可选列表**：`redbook/constants.js` 中 `TEMPLATE_OPTIONS`（素雅底纹、大图纯享、简约白、默认）。

### 原则

- 与选题、内容风格一致；避免过于花哨导致信息弱化。

---

## 五、小红薯运营专家 (Xiaohongshu Operations Expert)

**目标**：从**运营与增长**角度写内容、定话题、控节奏，提升曝光与互动。

### 产出

- **运营向标题/钩子**：前 3 秒抓住注意力（数字、痛点、结果、反常识）。
- **话题与标签**：3–5 个与内容强相关且有一定流量的 # 话题，供发布时填入。
- **发布节奏建议**：时段、频率、系列化等（可选）。

### 原则

- 内容可读性与平台推荐逻辑兼顾；话题需与正文匹配，避免纯蹭热点。

---

## 六、发布专家 (Publish Expert)

**目标**：按 **[readbook/README.md](readbook/README.md)** 执行「打开创作页 → 填表（标题+正文≤1000字）→ 话题 → 模板 → 一键排版 → 发布」，**反复重试直至发布成功**。不得中途询问用户。

### 职责与产出

1. **确定文章路径**：若用户指定了文件（如 `posts/xxx.md`），使用该路径；若为刚生成的内容，先写入 `posts/` 再使用。
2. **标题**：必须使用**标题专家**产出的标题，通过 `--title "标题"` 传入脚本；**不解析** md 文件中的 `# 标题`。
3. **正文**：从文件读取正文（脚本不解析标题行）；超过 1000 字时由脚本截断或由内容/运营专家事先缩写。
3. **话题**：根据内容专家/运营专家建议的 # 话题，通过 `--topics 话题1,话题2` 传入。
4. **模板**：根据审美专家建议，通过 `--template 素雅底纹`（或 大图纯享、简约白 等）传入。
5. **执行流程**（以 readbook/README.md 为准）：
   - 打开创作页：`node scripts/chrome-control.js "https://creator.xiaohongshu.com/publish/publish?source=official&from=menu&target=article"`
   - 填表+话题+模板+排版+发布：`node redbook/cli.js --file <路径> --title "标题专家产出的标题" --topics 话题1,话题2 --template 素雅底纹 --format --publish`
6. **未成功时**：按 runbook 故障排除（如 `node redbook/cli.js --inspect`、调整 `redbook/dom-selectors.js` 或 `redbook/utils/*.js`），再次执行直至成功。
7. **不询问用户**：不产出「请用户执行」的命令；不询问「要不再试」「请手动发布」；必须持续重试直至成功。

### 前置条件

- **Chrome**：用户已用 `--remote-debugging-port=9222 --user-data-dir="/Users/mac/chrome-profiles/1"` 启动 Chrome 并登录小红书。
- **文章**：正文已落盘为 `posts/xxx.md`（**无需**在正文里写 `# 标题`）；**标题由标题专家产出**，发布时用 `--title` 传入。

### 发布专家检查

- [ ] 已阅读并按 **readbook/README.md** 执行
- [ ] 已代为执行打开创作页 + 填表+排版+发布（或一条龙脚本），未只输出命令让用户执行
- [ ] 未成功时已按 runbook 故障排除重试，**未中途询问用户**
- [ ] 反复执行直至发布成功

---

## 工作流建议

1. **先选题**：用「选题专家」产出受众、角度；与用户确认后再写正文。
2. **再标题**：用「**标题专家**」根据选题与正文专门产出一条发布用标题（≤20 字），**不解析**正文里的 `# 标题`。
3. **再内容**：用「内容专家」或「小红薯运营专家」按笔记体搭大纲并写正文（**正文≤1000 字**）；可分段生成。
4. **后风格**：用「风格专家」通读，统一语气、修正排版。
5. **审美**：用「审美专家」选定模板；运营专家给出 3–5 个 # 话题。
6. **发布**：用「发布专家」执行 `redbook/cli.js --file <路径> --title "标题专家产出的标题" --topics ... --template ... --format --publish`，反复直至成功。

若用户只提「帮我写一篇关于 xxx 的小红书笔记」，可依次执行：选题 → **标题专家产出标题** → 内容（或运营专家）→ 风格 → 审美（模板+话题）→ 发布。
